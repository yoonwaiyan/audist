# Contributing to Audist

Thank you for your interest in contributing to Audist!

## Contributor License Agreement (CLA)

Because Audist uses a dual-license model (GPLv3 community + future commercial edition), all contributors must sign a **Contributor License Agreement (CLA)** before their pull request can be merged.

The CLA grants the project owner the right to include your contribution in both the open source and commercial editions of Audist. You retain copyright of your contribution — the CLA does not transfer ownership, it just grants a license.

### What you need to sign

- **Individual contributors** — sign the [Individual CLA](./CLA_INDIVIDUAL.md)
- **Contributing on behalf of a company** — your company must sign the [Corporate CLA](./CLA_CORPORATE.md)

### How to sign

CLA signing is handled automatically when you open a pull request. A bot will comment on your PR with a link to sign. You only need to sign once.

If you have questions about the CLA, [open an issue](https://github.com/yoonwaiyan/audist/issues).

## Getting started

1. Fork the repository and create a branch from `main`
2. Make your changes
3. Ensure the project builds and tests pass (`npm run typecheck && npm run test:e2e`)
4. Open a pull request

## Code style

- TypeScript is required for all source files
- Run `npm run lint` and `npm run format` before submitting
